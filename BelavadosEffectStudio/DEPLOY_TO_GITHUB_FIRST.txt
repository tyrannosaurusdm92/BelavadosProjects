BELAVADÖS EFFECT STUDIO — GITHUB DEPLOY READY

This package is flattened for GitHub Pages. Upload the CONTENTS of this folder into your existing GitHub Pages folder, not the folder itself.

Most likely reason local desktop shows v2 but GitHub shows v1:
1. GitHub Pages is still serving the old index.html from the configured Pages source folder; or
2. The new zip was uploaded as a nested folder instead of replacing the live index.html; or
3. The browser/PWA service worker cached the old v1 shell.

Deploy steps:
1. In your GitHub repo, open the folder that GitHub Pages serves for this app.
2. Delete the old app files in that folder first: old index.html, old assets, old subtools/mobile-pwa, old sw.js, old manifest.
3. Upload every file/folder from this deploy package directly into that same folder.
4. Commit changes.
5. Open github-cache-reset.html once from the live GitHub Pages URL, then it will clear old browser caches and redirect to index.html.

Verification:
- index.html is the only app layout.
- github-cache-reset.html is only a cache cleanup helper, not another app layout.
- No subtools/mobile-pwa folder is included.
- No v1 HTML source files are included in this deploy package.
- Top bar should say: v2 simple shell · GitHub deploy · mobile first
