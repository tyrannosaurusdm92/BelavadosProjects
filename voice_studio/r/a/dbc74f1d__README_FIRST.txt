Singular Accent Atlas

What this archive contains
- 00_DOCS: source readmes/licenses/manifests plus generated audits and manifests.
- 01_ACCENT_REGION_ATLAS: every non-doc source file preserved and categorized by accent/region/source.
- 02_GENDER_IDENTITY_SORT: the requested identity taxonomy, with speaker indexes where the uploaded sources exposed labels.

Preservation
- Preserved original source files: 1245
- Source archives merged: 12
- No source files were intentionally omitted.
- File contents were copied byte-for-byte into Windows-safe paths and verified with SHA-256 in FILE_MANIFEST_ALL_PRESERVED_FILES.csv.
- Original deep repository paths were flattened into safe filenames so each category stays shallow and usable on Windows.
- Original paths are retained in the manifest so the original tree can be reconstructed if needed.

Gender identity sorting
The uploaded datasets mostly use speaker sex/gender labels such as male/female or M/F. They do not explicitly prove the 14 identities requested.
For that reason:
- Cis-Female and Cis-Male folders contain cautious indexes based only on source-provided female/F and male/M labels.
- Other identity folders are present and include Belavados routing references where known.
- I did not infer identity from names, voice, accent, pitch, country, or file content.

Windows/path safety
- Forbidden Windows filename characters were replaced.
- Reserved Windows device names were avoided.
- Category paths are shallow: category/source/file.
- Long filenames were shortened with a hash suffix while preserving the original path in the manifest.

Generated files
Generated split metadata/index files are included for navigation and sorting. They do not replace the preserved originals.
